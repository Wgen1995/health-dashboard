# Health Dashboard Skill 设计规格

## 概述

为 opencode 创建一个 skill（SKILL.md），核心目标是**健康管理态势感知**——通过分析多维度 Excel 数据，自动生成交互式 HTML 看板 + Markdown 分析报告，帮助管理者全面感知各地医生投入、疾病分布、风险变化、医疗行为类型的整体局势与变化趋势。

## 触发条件

用户提到健康数据/医疗看板/Excel 分析/态势感知/生成报表等场景时自动匹配。
适用的关键词：健康数据分析、医疗看板、态势感知、医生投入、疾病对比、趋势分析、Excel 数据、生成看板、可视化报表等。

不适用：需实时数据库接入、非 Excel 数据源。

## 核心工作流

### Step 1 - Excel 扫描与语义理解
- 扫描指定路径或目录下的 `.xlsx`/`.xls` 文件
- 对每个文件：读取列名 + 前 N 行数据值
- AI 逐列做语义分析：
  - 列名字面含义（中英文）
  - 数据内容的实际值特征
  - 判断该列是维度列还是度量列
- 输出识别结果给用户确认

**参考语义词典（仅作参考，不写死）：**
- 医生/医师/doctor/接诊医生/咨询医生/随访医生 → 人员维度
- 所属区域/所属国家/region/country/location → 地理维度
- 疾病/病种/初步诊断/咨询内容/诊断 → 分类维度
- 风险等级/risk/level/risk_level → 分级维度
- 时间/日期/接诊时间/咨询时间/随访日期/最近一次监测时间 → 时间维度
- 接诊类型/咨询类型/咨询方式/随访方式 → 行为类型维度
- 次数/count → 频次度量
- BMI/体温/血压/脉搏/血氧/血糖/胆固醇/各项检验指标 → 数值度量

### Step 2 - 动态看板决策

AI 以**态势感知**为导向，根据 Step 1 的语义分析自主决定：
- 分组/聚合维度
- 适合的对比分析
- 时间序列趋势
- 度量统计方式（求和/均值/占比/排名）
- 异常波动点提示
- 至少生成 3-6 个视图
- 向用户展示看板方案说明选择的依据

### Step 3 - 生成看板与报告
- AI 编写 Python 脚本（pandas + plotly）
- 生成自包含交互式 HTML 看板
- 生成 Markdown 分析报告
- 每个视图附带文字解读

## 多文件处理

- 自动扫描目录下所有 Excel 文件
- 对每个文件独立做语义分析
- 判断文件间关系：
  - 结构相似 → 可横向合并比较
  - 不同医疗类型 → 按类型分组 + 跨类型对比
- 如果包含 4 种医疗类型维度 → 生成类型横向对比视图
- 先输出文件扫描结果让用户确认归类

## 大文件处理

- 千行级数据：先读前 100 行做结构理解
- 全量聚合在 Python 脚本内执行
- AI 审查聚合结果而非逐行阅读

## 输出规范

### HTML 看板
- 自包含 `.html`，可离线打开
- 每个图表有标题和文字解读
- 图表类型自主选择（柱/折线/饼/堆叠/热力等）
- 配色统一，布局清晰

### Markdown 报告
- 数据分析摘要（量级、覆盖范围、关键发现）
- 每个视图的对应表格与解读
- 趋势总结与建议

## 异常处理

- 无数值列 → 自动统计频次作为度量
- 无时间列 → 跳过趋势，只做静态对比
- 单维度 → 降级为统计汇总
- 编码错误 → 轮询 utf-8/gbk/latin1
- 含义不明确 → 列出前 5 个唯一值向用户询问
