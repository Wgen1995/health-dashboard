# 反幻觉约束全集（ANTI_HALLUCINATION）

> 适用范围：所有 SourceCPT 子技能 + 共享规范引用。
> 约束力：**强制**。LLM 违反任一条目即触发对应 Phase 的判定降级或拒收。
> 总条数：56 条（10 全局 + 46 Phase 专属，其中编号 12-14 为旧 grep 版占位已由 15-17 取代）。

---

## 反幻觉设计哲学

LLM 比传统 AST 扫描器更易"看似合理"地编造漏洞证据。SourceCPT 防误报防漏报的核心是**结构性榨出证据**：把 VULN 判定变成"举证责任"——LLM 必须提出六项验收证据，不是"我觉得这像漏洞"。

约束分两类：
- **禁令**（不准 X）：反幻觉#1-#10 全局、#15-#41 Phase 专属行为禁令
- **举证**（必须 Y）：#42-#59 与证据完整性/产出格式相关

违反即对应 Phase 内降级/disprove/拒收/阻塞报告。

## 与 Loop 工程的关系

部分约束（#30、#38、#40）借鉴 loop engineering 的"拆卷子原则"（判卷师与拆卷师分开），但**主体不 loop**——安全审计的鉴权与支付逻辑不该让 loop 碰，只在对抗仲裁等局部 ≤3 轮应用。详见 LOOP_POLICY.md。
