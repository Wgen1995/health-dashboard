# 审计深度分层（OFFSET_RATING）

> 适用范围：所有 Phase 在标注执行深度时使用 T0-T3 分层。
> 对照：GenCPT L0-L3 是黑盒运行时语义（SSH 中观察/容器内验证），T0-T3 是白盒语义（读代码/不执行命令）。两者本质不同，不存在等同关系。
> 约束力：**强制**。每 WU 输出必标深度层级，便于报告溯源与 QA 校验。

---

## T0 — 单文件模式匹配

**执行方式**：Glob/Grep + Read ±20 行（仅截命中行附近，不读完整文件）
**用途**：识别入口面、sink 索引、grep 模式命中、全局鉴权配置识别
**Phase 使用**：0、2b 第 1 层、1b 配置识别
**产出**：节点清单、raw 命中行（路径+行号+片段）
**反幻觉**：每命中必引 bash grep 原始输出（#1、#2）

---

## T1 — 跨函数污点追踪

**执行方式**：Read sink 所在函数 + 沿调用链反查 2-5 跳（每跳读 ±20 行）
**用途**：source → propagation → sink 完整路径追踪，建立 evidence_chain
**Phase 使用**：2a、3、4a、5 链节点验证
**产出**：evidence_chain 中的 propagation 数组，每跳 `{from, to, file, line, transform_type}`
**反幻觉**：每跳必有 Read 证据，禁凭记忆推断（#20）

---

## T2 — 跨模块可达性验证

**执行方式**：Read 跨模块文件 + 临时方法级图谱展开（SRC_ACCESS §4）
**用途**：链节点跨模块可达性、鉴权配置覆盖判断、业务约束对照
**Phase 使用**：4b、5、6 挑战者验证
**产出**：可达性判定（confirmed_full/confirmed_partial/theoretical）、节点间可达证据
**反幻觉**：跨模块 Read 留底，禁用上轮记忆作证据（#38 对抗 loop 每轮新 Read）

---

## T3 — 业务逻辑推断

**执行方式**：Read 业务文档(若有 1c 启用) + 代码深度 Read + LLM 推理
**用途**：业务逻辑漏洞（状态机偏离、权限矩阵不符、价格篡改）、鉴权逻辑推断
**Phase 使用**：4b 业务漏洞推理、6 挑战者业务层挑战
**产出**：deviated 候选（业务逻辑漏洞）、推理证据链
**反幻觉**：文档可信度分级（#42），文档约束不许单独判 VULN 必配代码证据

## 与 GenCPT L0-L3 语义对照

| GenCPT 黑盒（运行时） | SourceCPT 白盒（静态代码） | 本质差异 |
|---------------------|---------------------------|---------|
| L0 宿主机观察（SSH，只读命令） | T0 单文件模式匹配（本地 Glob/Grep） | 不远程 / 不执行只读盘 |
| L1 容器内观察（kubectl exec 进入容器） | T1 跨函数污点追踪（Read 调用链） | 不进入运行时只读代码 |
| L2 容器内攻击验证（差分证明 + payload 执行） | T2 跨模块可达性（Read 邻居+临时方法图） | 不部署 payload 复现 / 路径成立即可判定 |
| L3 理论验证（破坏性操作不执行） | T3 业务逻辑推断（语义推理） | 白盒强项，黑盒无能 |

**关键差异**：
- L0-L3 是"在哪观察"的语义，T0-T3 是"读到多深"的语义
- 黑盒 L2 需实际执行做差分，白盒 T2 路径成立即判可达
- 黑盒 L3 是兜底（破坏性操作禁执行的妥协），白盒 T3 是强项（业务推断是 LLM 优势）
