# 健康管理态势感知看板

## 概述

本 skill 指导 AI 对员工健康管理 Excel 数据**全自动态势感知分析**：
1. 自动扫描 Excel 文件，LLM 语义理解数据列含义
2. 自主决策分析维度和看板方案
3. 直接生成自包含交互式 HTML 看板 + Markdown 分析报告

**核心原则：**
- 不预设表格结构，AI 必须从实际数据中语义识别列含义
- AI 全自主决策，**默认不询问用户**，直接产出结果
- **仅在真正无法确定某列含义时**（列名和值都无法匹配参考词典），才列出该列前 5 个唯一值向用户提问
- 对识别出的列和看板方案，在生成过程中以简讯形式告知用户即可，无需等待确认

---

## 何时使用

**触发场景：**
- 用户要求分析健康/医疗 Excel 数据
- 用户要求生成健康管理看板、态势感知报告
- 关键词包含：健康数据、医疗看板、态势感知、医生投入、疾病对比、趋势分析、Excel 分析、数据看板、可视化报表、健康管理、门诊记录、健康咨询、风险人群、随访记录

**不适用场景：**
- 数据在数据库中而非 Excel 文件
- 非健康/医疗领域的数据分析
- 只需要简单的文件格式转换不涉及分析

## 核心工作流

### 交互流程

AI 执行本 skill 时，必须遵守以下交互节奏，不得跳过用户确认步骤：

```
用户提供 Excel 文件路径/目录
    │
    ▼
AI 扫描文件，展示文件列表
    │
    ▼  ─── 用户确认文件列表
    │
    ▼
AI 读取每个文件结构，语义分析每列含义
    │
    ▼  ─── 输出列识别报告，用户确认或纠正
    │
    ▼
AI 决策看板方案，展示视图清单
    │
    ▼  ─── 用户确认看板方案
    │
    ▼
AI 编写并执行 Python 脚本，生成 HTML + Markdown
    │
    ▼
AI 自检验证，输出给用户
```

**关键原则：**
- 用户提供路径后，AI 必须先展示文件扫描结果
- 语义理解结果必须先给用户确认
- 看板方案必须先给用户确认
- 只有用户明确确认后才能执行生成

### Step 1：Excel 扫描与语义理解

**1.1 文件扫描**
- 扫描用户指定的路径（文件或目录）下所有 `.xlsx` 和 `.xls` 文件
- 向用户展示扫描结果：发现几个文件、文件名、文件大小
- 让用户确认是否全部纳入分析

**1.2 逐文件结构探测**
- 对每个文件，使用 Python `pandas` 读取前 100 行数据
- 提取：列名列表、每列的前 5 个唯一值示例、每列的数据类型
- **禁止直接打印全部原始数据**——只需列名 + 数据样例 + 类型信息

**1.3 语义理解每列含义**
对于每一列，AI 必须通过以下两步判断其含义：

(a) **列名字面分析**：根据列名（中英文）推断可能的业务含义
(b) **数据内容验证**：检查该列的实际数据值是否与 (a) 的推断一致
  - 分类维度特征：有限个唯一值（如"高/中/低"、"男/女"、国家名）
  - 数值度量特征：连续数值范围（如 BMI 指数、血糖值、血压值）
  - 人员维度特征：中文姓名或 ID 编号
  - 时间维度特征：日期/时间格式、或在时间序列上单调递增

**1.4 输出列识别报告**
向用户展示一份列识别报告，格式为表格：

| 列名 | 推断含义 | 类型（维度/度量） | 可选值示例 |
|------|---------|------------------|-----------|
| 接诊时间 | 时间维度（就诊时间） | 维度 | 2025-01-15, 2025-01-16... |
| 接诊医生 | 人员维度（医生姓名） | 维度 | 张医生, 李医生... |
| BMI指数 | 数值度量（BMI） | 度量 | 18.5, 22.3, 27.1... |

然后询问用户："以上列含义识别是否正确？是否需要调整？"

### Step 2：动态看板决策

基于 Step 1 的列识别结果，AI 以**态势感知**为导向，自主决定生成哪些看板视图。

**2.1 看板决策规则**
根据识别出的维度列和度量列，AI 应综合考虑以下视角：

- **人员维度分析**（如有医生/人员姓名列）：
  - 医生投入看板：每位医生的工作量（记录数）、覆盖的患者数、覆盖地区数、参与的医疗类型数
  - 工作量时间趋势：每位医生按月/季度的投入变化
  - 排名对比：投入 Top N 和 Bottom N 医生

- **地理维度分析**（如有区域/国家/机构列）：
  - 地区分布图：各地区/国家的记录数、患者数、医生数分布
  - 地区对比：各地区关键健康指标的均值对比（如平均 BMI、血糖异常率等）
  - 地区趋势：各地区的时间序列变化

- **分类维度分析**（如有疾病/诊断/风险等级/医疗类型列）：
  - 分类构成：各类别的数量分布和占比（饼图或柱状图）
  - 分类趋势：各类别的时间变化趋势
  - 分类交叉分析：疾病类型 × 风险等级、医疗类型 × 地区等交叉对比

- **时间维度分析**（如有时间列）：
  - 整体工作量时间趋势（按月/季度总记录数变化）
  - 各维度的上升/下降趋势分析
  - 增长率计算（环比/同比）
  - 异常波动点标注

- **数值度量分析**（如有体征/生化指标列）：
  - 各项指标的统计描述（均值、中位数、标准差、异常率）
  - 指标的时间趋势：各项指标均值随时间的变化
  - 指标的地区/医生差异

**2.2 看板方案输出**
AI 综合以上规则，输出一份看板方案给用户确认：

```
## 看板方案

基于本次数据分析，建议生成以下看板视图：

| 编号 | 看板名称 | 使用维度 | 图表类型 | 说明 |
|------|---------|---------|---------|------|
| 1 | 医生投入排行榜 | 医生、记录数 | 横向柱状图 | 展示Top医生工作量 |
| 2 | 地区分布对比 | 区域/国家、记录数 | 柱状图/地图 | 各地区服务量对比 |
| 3 | ... | ... | ... | ... |

如确认请回复"开始生成"，或指定需要调整的视图。
```

**2.3 最少视图要求**
- 至少 3 个视图，不超过 8 个（避免信息过载）
- 每个视图必须有明确的态势感知目的
- 如果数据不支持某类分析，说明原因并跳过

### Step 3：生成看板与报告

用户确认看板方案后，AI 编写 Python 脚本执行数据聚合与图表生成。

**3.1 Python 脚本要求**
- 使用 `pandas` 读取和聚合数据
- 使用 `plotly` 生成交互式图表
  - 优先使用 `include_plotlyjs='cdn'` 以保证 HTML 文件轻量
  - 如用户要求纯离线或网络受限，使用 `include_plotlyjs=True` 嵌入完整 plotly.js
- 脚本必须一次性生成完整的 HTML 文件和 Markdown 报告
- 所有数据处理在 Python 脚本内完成，AI 不逐行审查原始数据

**3.2 HTML 看板要求**
- 单个自包含 `.html` 文件
- 布局：顶部标题栏 + 下方多图表网格排列
- 每个图表区域包含：
  - 图表标题
  - 交互式图表本体（可缩放、悬停显示数值）
  - 2-3 句文字解读（数据洞察）
- 配色统一、专业，推荐使用 plotly 内置主题
- 图表类型根据数据自主选择：柱状图（对比）、折线图（趋势）、饼图（构成）、堆叠柱状图（分层对比）、热力图（交叉分析）、散点图（相关性）

**3.3 Markdown 报告要求**
保存为 `.md` 文件，与 HTML 同目录，内容包含：
- **数据概览**：总记录数、覆盖时间范围、覆盖国家/地区数、医生数
- **看板分析**：按视图逐一输出关键数据表格 + 文字解读
- **态势总结**：3-5 条关键发现和建议
- **数据说明**：数据来源文件、分析时间

**3.4 多文件处理**
如果扫描到多个 Excel 文件：
- 先判断文件之间的关系：
  - 列结构相同或高度相似 → 可能是同一类数据的不同批次，可合并
  - 列结构明显不同 → 属于不同数据类型，各自独立分析
- 如果发现 4 种典型表格（门诊、咨询、风险、随访），应生成跨类型对比视图
- 先向用户展示文件归类和合并方案，确认后再执行

**3.5 大文件处理**
- 读取前 100 行做结构理解（已在 Step 1 完成）
- 全量聚合在 pandas 中执行：groupby、agg、describe 等
- 聚合结果（而非原始数据）输出给 AI 用于生成文字解读
- 如文件过大（>100MB）导致读取缓慢，先提醒用户可能需要较长时间

## 异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| Excel 文件无法打开 | 尝试不同引擎（openpyxl、xlrd）和编码（utf-8、gbk、utf-8-sig、latin1） |
| 列名无法理解 | 列出该列的前 5 个唯一值，向用户询问含义 |
| 全部分类列（无数值度量） | 以"频次"（记录数）作为度量值进行统计 |
| 无时间列 | 跳过趋势分析，仅输出静态分布和对比 |
| 全部时间列为同一时间 | 降级为静态分析，说明无法计算趋势 |
| 只有单一维度可分析 | 输出统计汇总视图（频次分布表+饼图） |
| 数值列含无效值 | 自动过滤 NaN、Inf、明显异常值（如 BMI>100），并在报告中说明过滤了多少行 |
| 脚本执行失败 | 输出错误信息给用户，提供手动修复建议 |
| 文件编码导致中文乱码 | 依次尝试 utf-8-sig → gbk → gb2312 → latin1，选择能正确读取中文的编码 |

## 验证检查清单

在看板生成后，AI 必须自检以下各项：

- [ ] 是否确实从实际文件读取了列名和数据样例？（不能凭空猜测）
- [ ] 是否在看板生成前向用户确认了列识别结果？
- [ ] 是否在看板生成前向用户确认了看板方案？
- [ ] HTML 文件是否能独立打开（不依赖服务器）？
- [ ] 每个图表是否都有标题和简短文字解读？
- [ ] Markdown 报告是否包含数据概览、态势总结？
- [ ] 是否有多余的原始数据被输出到报告中？（应尽量避免）
- [ ] 异常数据（过滤的行）是否在报告中说明？
